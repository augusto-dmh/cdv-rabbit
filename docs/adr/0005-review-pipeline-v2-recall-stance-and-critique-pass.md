# Review pipeline v2: recall stance, two-call draft → critique, same-provider critic

Phase 6 production review of DocInt PR #35 (8 files, 233/-110 across 3 orthogonal refactors) returned zero inline Findings and a "no significant issues found" wrap-up — a clear silence-bias failure of `review_v1.txt`. v2 reverses the stance from "prefer silence" to **high-recall structured output** modeled on CodeRabbit (Walkthrough + tiered Findings + collapsed Nitpicks) — see CONTEXT.md for the locked domain terms. The schema bumps to `review_result_v2.json` alongside v1 (v1 stays frozen for AC23 snapshot); a `workspaces.review_schema_version` enum (`v1|v2`, default `v1`) gates per-Workspace rollout. v2 is authored to the **OpenAI strict intersection** (every property `required`, optionals encoded as `{ type: ['string','null'] }`) so a single schema file feeds both Anthropic strict tool use (plan §3.0.3) and OpenAI strict structured output (plan §3.6) without per-driver variants — keeps the `LlmDriverInterface` flat surface principle from ADR 0004. The pipeline becomes **two-call draft → critique, same-provider**: `LlmDriverInterface` gains `critiqueDraft(DraftReviewDto): CritiqueResultDto`; only Findings approved by the critic reach `CommentPoster`. The critic uses the same provider as the drafter so the per-provider cost-reservation Lua keys (`workspace:{id}:tokens:{date}:{provider}`) and per-Workspace DPA mapping stay intact; cross-provider critiquing is rejected for MVP despite stronger empirical quality because it doubles the env/DPA/cost surface. The system prompt (`review_v2.txt`) embeds **5 positive + 2 negative Laravel-flavored few-shot exemplars** wrapped in `<example>` tags per Anthropic prompting guidance — chosen over abstract negative instructions ("do not invent findings", "prefer silence") that caused the v1 bias. Cost ceiling absorbs the ~2× call increase via a `cost_per_review_factor` config; no schema change. **Quality is gated by a new eval harness** (`tests/Eval/golden/` with 10 hand-curated PRs from DocInt, cdv-rabbit, and intranet-cdv; `php artisan rabbit:eval` runs every (provider × schema_version) cell and computes recall/precision against `expected_findings.json` using an **LLM-as-judge with rotated provider** — Claude drafts judged by GPT and vice versa, eliminating same-family sycophancy). v2 prompt changes do not merge without delta vs the golden baseline — added as a locked contract in AGENTS.md §4. Rollback is per-Workspace: flip `review_schema_version` back to `v1`; no migration needed.
