# GitHub App is the auth model for the GitHub SCM driver

The GitHub SCM driver authenticates via the GitHub App installation flow — app-level RSA private key in env, per-Workspace `github_installation_id` — rather than fine-grained or classic PATs. Reasons: a stable non-human bot identity (matching the CodeRabbit/Dependabot pattern), per-org granular install scopes, and a native install/uninstall UI that doubles as the v0.3 self-service onboarding flow promised in `AGENTS.md §8`. Trade-offs: ~3-5 days extra implementation vs PAT, an asymmetric `ScmDriverInterface` (`registerWebhook` is a no-op on GitHub because the App's webhook URL is set once and auto-fires for every installed repo), and a shape change in the `AGENTS.md §4` "HMAC webhook auth" locked contract — secret is per-Workspace for BB, per-App for GitHub.
