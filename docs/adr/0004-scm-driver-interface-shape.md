# ScmDriverInterface: flat wide surface with normalized DTOs

The SCM abstraction is a single `ScmDriverInterface` (no segregated capability interfaces) returning normalized DTOs (`RepositoryDto`, `PullRequestDto`, `FileChangeDto`, `CommentHandle`, `WebhookHandle`, `CredentialCheck`) — provider-shaped JSON arrays die at the driver boundary. Asymmetric methods (`registerWebhook`, `deleteWebhook`) return `?WebhookHandle` / `void` with PHPDoc documenting the GitHub no-op semantics. The choice favours a small, mechanically-callable surface for the four call sites (`ConnectController`, `RepositoryController`, `CommentPoster`, `ReviewPullRequestJob`) over a strictly-correct OO segregation (separate `SupportsPerRepoWebhooks` etc. interfaces) that would only pay off at 5+ providers. `BitbucketClient` is renamed and reshaped to `BitbucketDriver implements ScmDriverInterface` in `app/Services/Scm/BitbucketDriver.php`; `GithubDriver` joins it; `ScmDriverFactory::make(Workspace)` mirrors the existing `LlmDriverFactory`. The `app/Services/Bitbucket/` directory is removed.
