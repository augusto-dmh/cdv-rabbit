# Agent Prompt: per-Finding CodeRabbit-style instruction for coding agents

Builds on ADR 0005 (review-pipeline-v2). Each `findings[]` entry gains an `agent_prompt` field — a natural-language imperative rendered as a collapsed `<details>🤖 Prompt for AI Agents</summary>` block at the footer of the inline comment — so a developer can copy it into a third-party coding agent (Claude Code, Cursor, etc.) and have the fix applied without re-reading the PR. The field is added to `config/cdv-rabbit/schemas/review_result_v2.json` as `{ "type": ["string","null"], "maxLength": 1500 }` under the OpenAI-strict intersection (every property `required`, optional encoded as a nullable union) so the single schema continues to feed both Anthropic strict tool use and OpenAI strict structured output without per-driver variants — keeps the flat-surface principle from ADR 0004. The draft LLM emits the body only (e.g. `` `In \`@app/Foo.php\` around lines 40 - 44, ...` ``); the fixed preamble `"Verify each finding against the current code and only fix it if needed."` is concatenated in PHP by `CommentPoster::postInlineFinding()` so future preamble edits (i18n, branding, instruction tuning) do not pay the `rabbit:eval` golden-baseline gate. The same `LlmDriverInterface::reviewDiff()` call that produces `message` and `suggestion` produces `agent_prompt` — a separate post-draft call was rejected because the existing pipeline already runs draft → critic and adding a third call would push p95 latency from ~75s to ~110s (already a tension with consumer-repo auto-merge per AC51) and would re-pay input tokens that prompt caching has already amortized on the draft call. The same-provider critic from ADR 0005 verdicts the Finding as a whole; the `agent_prompt` rides on the verdict — approve keeps it, reject drops it. The critic prompt does NOT learn to verdict `agent_prompt` separately, preserving symmetry with how it already treats `message` and `suggestion` (anchor-only validation, trust-the-content). To avoid bloating critic input tokens for content the critic does not judge, `ReviewPullRequestJob::handle()` strips `findings[].agent_prompt` from the draft JSON before `critiqueDraft()` and reattaches the original strings to approved Findings before calling `CommentPoster::postV2()`. Rule-based path-validation of the prompt (parsing `@<path>` and cross-checking the diff file list) is explicitly **out of scope** for MVP — promoted to v0.2 if the pilot surfaces hallucinated-path prompts; the cost-benefit at MVP is unfavorable because the critic already gates the anchor and `message`/`suggestion` are not similarly validated, so introducing a per-field validator only for `agent_prompt` would be inconsistent. `CommentSanitizer::sanitize()` is updated globally (not via a separate `sanitizeAgentPrompt()` method) to preserve `@<path>` references — the regex changes from `/@[A-Za-z0-9._-]+/` to `/@(?![\w._-]*\/)[\w._-]+/`, stripping `@<username>` (no slash, real user mention) while preserving `@<path>` (slash-bearing, agent-readable file reference); the same improvement flows to `message`, `suggestion`, `walkthrough`, and `nitpick.message` as a side benefit. The rendering wrapper (`<details>` + `<summary>` + triple-backtick fence + body + closes) is assembled outside the sanitizer, matching the pattern already used in `CommentPoster::postV2Summary()` for the Nitpicks block; only the body content passes through `sanitize()`. Findings that fail to anchor inline (SCM 422 fallback per `tryPostInlineFinding()`) carry their `agent_prompt` into the summary's unresolved-Findings section as a nested `<details>` so the developer still gets the agent-ready instruction in the exact case where it matters most (issue can't be seen inline). Findings with `agent_prompt: null` render with no `<details>` block — clean comment, no empty-prompt cerimony. The schema change requires a `rabbit:eval` baseline run against the 10 golden PRs on both providers per the locked contract from ADR 0005; the prompt edits to `review_v2.txt` (new "Agent Prompt rules" section, `agent_prompt` field added to the 5 positive few-shot exemplars, "Forbidden content" amended to allow `@<path>` inside `agent_prompt`) gate on no-regression of recall/precision vs the previous baseline. The `cache_control: ephemeral` system-prompt cache is invalidated once at rollout (one Review pays full input tokens, ~$0.30 vs ~$0.03 cached); cache rebuilds on the next call. `cost_per_review_factor` may need to climb from 2.0 to ~2.3-2.5 to absorb the ~25k extra output tokens per Review (1500 chars × ~70% of 25 Findings populated); confirmed against the pilot's first week of telemetry, not pre-emptively bumped. Glossary entry `Agent Prompt` added to `CONTEXT.md`. The "Comment cap + AI label" row in `AGENTS.md` §4 is amended (not a new row) to pin the three invariants: preamble-in-PHP, footer position, path-aware sanitization.
